# 情侣复盘 Skill · v4.0

## 你会经历什么

```
① 上传记录  →  ② 补充信息  →  ③ 描述冲突  →  ④ 对抗式对话  →  ⑤ 论文复盘
   （5分钟）       （2分钟）        （3分钟）        （6分钟）          （5分钟）
                                    └─ 子代理A × 子代理B ─┘
```

**总时长：约20分钟 | 你需要：聊天记录 + 一个想复盘的冲突**

---

## 第一步：上传记录（推荐用 couple-wechat-importer）

### 方式 A（推荐）：微信数据库直读

如果你已有解密的微信数据库，我会调用 `couple-wechat-importer` skill：
- 从 `MicroMsg.db` 找到对方 wxid
- 从 `MSG0.db` 提取完整聊天记录
- 自动输出：**时间线 + 行为分析 + 依恋标记 + Markdown 报告**

你只需提供：联系人昵称（如"壮壮"）或数据库路径。

### 方式 B：直接粘贴

把微信聊天记录复制粘贴给我，格式不限，我能读懂。

### 方式 C：截图或描述

没有记录也没关系，直接描述冲突经过。

### 我会做什么

读取记录后，我会：
- 识别两个人各自的沟通模式（用词习惯 / 回复节奏 / 沉默方式）
- 找出情绪转折点
- 识别两个人各自的「行为特征」而非贴标签

---

## 第二步：补充信息

我会问你几个问题，帮助我更准确地还原冲突：

1. **这次冲突是什么时候发生的？**
2. **冲突的结果是什么？**（和好了 / 冷战中 / 还在僵着）
3. **你希望这次复盘解决什么问题？**

---

## 第三步：描述冲突

告诉我你最想复盘的**一个具体冲突**。

好的描述：
> "上周五晚上我说要加班，他连发了十几条消息问我和谁在一起，我没回，他第二天一整天没理我"

不好的描述：
> "我们沟通有问题"（太抽象，我没法分析）

---

## 第四步：对抗式对话（子代理）

基于你上传的真实数据，我会生成**两个子代理**：

- **子代理 A**：代表你（基于你的沟通模式和行为特征）
- **子代理 B**：代表对方（基于对方的沟通模式和行为特征）

两个子代理会模拟你们当时的对话，进行**6分钟左右的对抗式交流**。

每个子代理的「性格」来自你上传的真实数据：
- 不是「焦虑型 / 回避型」这种标签
- 而是「这个人在数据里是怎么说话的」

我会在对话进行到关键点时插入**理论注解**，帮你理解为什么对话会这样发展。

---

## 第五步：论文复盘总结

对抗式对话结束后，我会基于论文理论给出分析：

**Paper A：追逐-逃避循环**
- 你们的数据里是否出现了追逐-逃避循环？
- 循环在哪个节点启动？在哪个节点断裂？

**Paper B：结构性冲突**
- 这是沟通问题，还是系统设计冲突？
- 双方各自的情感资源模型是什么？

**Paper C：EWDM动态模型**
- E（期望）在这段对话里是怎么变化的？
- 打破循环的节点在哪里？

**输出：**
- 综合评分（满分10）
- 核心矛盾
- Top3踩雷点
- 具体可执行的改进方案（最多3条）
- 动态黑名单更新

## 技术说明

### 子代理对话机制

- **主控**：Hanako（我）是主控，协调两个子代理的对话节奏
- **子代理 A**：代表方A，在对话中进行攻击和防御
- **子代理 B**：代表方B，在对话中进行攻击和防御
- **理论注解**：我在关键节点插入论文框架注解
- **对话长度**：约6分钟（6-10轮对话，视冲突复杂度而定）

### 素材处理原则

- **不贴标签**：不用「焦虑型」「回避型」这种词来形容用户
- **还原行为**：用数据说话——"他在数据里是怎么说话的"
- **严谨分析**：每个结论都附上论文依据

### 数据来源

本 skill 内置以下理论框架：
- Bowlby 依恋理论
- Ainsworth 依恋类型分类
- Hazan & Shaver 成人依恋迁移
- Gottman 关系修复研究
- Paper A/B/C（免费可下载全文）

## 代理定义

### 代理1：笔杆子（记录员）
- 客观记录对话/事件时间线
- 不评判，只陈述事实
- 输出：《复盘事件时间线》

### 代理2：参谋（批评员）
- 从对方视角发起质疑和攻击
- 覆盖五个维度：情绪控制 / 沟通质量 / 踩雷清单 / 修复态度 / 关系增益
- 每条批评附上"杀伤力评分（1-10）"

### 代理3：裁判（打分员）
- 综合双方立场，给出客观评分（满分10）
- 6分以下触发"关系预警"
- 输出：《复盘报告》

## 复盘流程（完整8步）

### Step 0 — 素材导入
读取用户数据，识别两个人的真实沟通模式。

### Step 1 — 信息确认
确认冲突时间、结果、用户目标。

### Step 2 — 冲突描述
用户描述具体冲突场景。

### Step 3 — 时间线重建
笔杆子输出《复盘事件时间线》：
```
[时间] [事件] [方A反应] [方B反应] [结果]
```

### Step 4 — 对抗式对话（核心步骤）
生成两个子代理，基于真实数据模拟冲突对话。
- 子代理性格来自数据，不是预设标签
- 每个关键节点插入论文理论注解
- 约6分钟，6-10轮

### Step 5 — 参谋攻击
参谋按五个维度逐一击穿：

| 维度 | 攻击问题 |
|------|---------|
| 情绪控制 | 情绪失控是否属于惯常反应？ |
| 沟通质量 | 双方是否在同一层面（事实vs感受）？ |
| 踩雷清单 | 这句话是否触发了对方雷点？ |
| 修复态度 | 冲突后是否有主动修复？时机合适吗？ |
| 关系增益 | 这次冲突有建设性产出吗？ |

### Step 6 — 裁判打分

| 维度 | 权重 |
|------|------|
| 情绪控制 | 20% |
| 沟通质量 | 25% |
| 踩雷清单 | 20% |
| 修复态度 | 20% |
| 关系增益 | 15% |

### Step 7 — 论文总结 + 结论输出

基于 Paper A/B/C 给出：
- 综合评分
- 核心矛盾
- Top3踩雷点
- 具体改进方案（最多3条）
- 动态黑名单更新
